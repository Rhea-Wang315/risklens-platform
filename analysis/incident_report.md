## Operational Implications

1. Decision automation can reduce loss exposure for high-confidence attack patterns.
2. `WARN` workflow can shorten analyst response loops if integrated with alert routing.
3. Labeled-data quality directly impacts confidence in false-positive estimates.

## Assumptions and Limitations

1. Prevented loss is estimated using `subsequent_transactions_volume_usd`, not realized PnL.
2. Time-saved estimate assumes fixed manual review duration per `WARN`.
3. Historical replay does not model market impact or adversary adaptation.

## Recommendations

1. Replace sample dataset with production whale-sentry exports.
2. Calibrate prevented-loss model with compliance and operations teams.
3. Add protocol-specific cost and impact factors for higher-fidelity ROI estimates.
4. Version-control analysis outputs for each demo/interview snapshot.
